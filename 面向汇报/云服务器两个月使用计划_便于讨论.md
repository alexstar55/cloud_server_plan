## 1) 目标（Outcome）
两个月内，建成“仿真/生成数据 → NuScenes导出 → 一致性校验 → 混合训练验证 → 切片报告”的闭环，支撑 **robobus 感知半径 50m → 80m**，重点缓解 **远距（50–80m）与 `bus` 类漏检**；同时控制近距车辆偶发误检风险。

> 指标口径：离线以 **mAP / NDS** 为主；**召回/误检率**在路测侧获取，作为参考与一致性验证（不作为本计划主验收指标）。

---

## 2) 资源与硬约束（决定可行性边界）
- 服务器：双 5880 GPU（显存/驱动以 `nvidia-smi` 为准），带可视化界面；使用期 2 个月  
- 网络：云端可联网但**不可直连国外网络**；外部依赖（镜像/权重/数据）采用 **本地下载（VPN）→ 上传云端 → 离线导入**
- 被服务模型与评测：**BEVFusion（训练MIT版本/部署cuda-bevfusion）+ CenterPoint（MMDet3D）**；离线主指标 **mAP/NDS**
- 数据/传感器：必须 **6cam + 1lidar**，内外参/时间同步与实车一致；导出 **NuScenes可读**（含ego pose，支持sweep）
- 业务约束：robobus 不做夜间；有**遮挡标签**，无雨夜标签；现有范围50m，目标100m；近距车辆偶发误检、远距`bus`更易漏检
- 训练策略：真实:仿真 ≈ **1:3+**，并对 **远距切片过采样**

---

## 3) 验收标准（Deliverables｜写死）
**D1 数据（可训练）**  
- ≥ **1000 sequences**（6cam+1lidar），NuScenes格式可读  
- 通过一致性校验（时间戳/位姿连续、sweep可用；导出版本有manifest：生成参数/切片标签/版本号）

**D2 指标（离线 mAP/NDS）**  
- 远距切片（50–80m）mAP/NDS **稳定提升**（关键指标提供 bootstrap CI 判断稳定性）  
- 全量 NDS **守门线**：不低于基线 -0.2（阈值可结合基线再校准）  
- `bus` 单类报表：明确改善/不改善结论与原因归因（场景/切片）

**D3 工程资产（可复用/可交接）**  
- 一键闭环：`generate → export → validate → report`  
- **自动化工具链**：沉淀可复用的预处理脚本（如 `calib_injector` 标定自动注入、`alpasim2nuscenes` 高性能流水线转换）。
- 可视化回放（GUI）：样例回放 + 失败案例清单 + 距离分桶/遮挡/`bus`切片报告模板

## 5) 平台策略（先闭环，后评测；可替换生成器接口）
- **优先**：用团队成熟仿真栈（如 CARLA/现有平台）先保证 W2 闭环与验收链路稳定  
- **并行评测（领导要求）**：alpaSim / cosmos / Alpamayo / omniverse 作为“生成器插件”接入；只要能导出统一中间格式并通过校验，即可纳入同一评测/报告体系  

### 5.1 生成器插件优先级与硬件调度（建议｜以两个月闭环交付为目标）
> 原则：先把“可落盘→可导出→可校验→可回放→可训练/可评测”的最短链路做出来，再扩平台覆盖面。

**【硬件调度战术：M×N 异构流水线并行】**
针对双卡无 NVLink 的硬约束，主线数据生成阶段**不跨卡并行单一任务**，采用流水线策略：
- **GPU 0**：专职跑 AlpaSim / Omniverse 渲染与物理引擎（吃显存）。
- **GPU 1**：专职跑 Cosmos Transfer 增强推理 + 数据后处理转换（转码/压缩）。
- **收益**：双卡互不等待，内存到显存直传，生成效率翻倍。

**【生成器接入优先级】**
1) **cosmos-transfer1-carla（Docker）优先纳入统一闭环**
   - 原因：容器形态明确，最利于离线转运与快速验收；先把“插件接口规范”跑通。
2) **Omniverse/Replicator（或 Isaac Sim）作为第二优先**
   - 原因：更贴近具身/物理与多传感器仿真范式；但链路更长，放在 W1–W3 并行推进，不阻塞 W2 验收。
3) **alpaSim / alpamayo**
   - 原因：依赖内部资产与接口成熟度；按“能否稳定导出 6cam+lidar+pose + sweep”决定插入顺序。

### 5.2 诊断与稳定性指标（低成本加分项：把健康度监控纳入闭环）
- 在 `validate` 与训练日志中增加“健康度指标”通道，作为：
  - exporter/pose/time 对齐问题的早期定位辅助
  - 混合训练不稳定的归因辅助
- **复用已在本地跑通的验证资产**：`面向个人执行/mve-training-warning/`（Sinkhorn/约束偏离度的诊断代码）
  - 验收形态：该算法已在本地验证有效。在云端只需将其作为轻量级监控脚本，接入到训练/校验日志中，即可产出稳定性诊断图表，**不额外占用云端开发排期**。

## 6) 主要风险与止损（保证两个月不失控）
- **网络限制**：外部依赖拉取失败 → 统一走离线转运（docker save/load 或内网registry）  
- **时间/位姿对齐风险（最大）**：W2 前若校验不过 → 先降级（1–2路相机+lidar）保闭环，同时并行修复6cam全量对齐  
- **平台集成风险（alpaSim/cosmos链路长）**：不影响主闭环；以“插件接入”推进，不重写导出/评测
