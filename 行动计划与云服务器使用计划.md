## 1) 两个月的“可验收”KPI（写给周会/OKR 的一句话）

### 1.1 主 KPI（必须完成）

在两个月内，搭建一个可自动运行的“仿真数据生成闭环”，针对远距目标切片生成数据并完成混合训练验证：

- 生成 **≥1000 组**可训练样本（6cam+1lidar，时间/位姿对齐，NuScenes 格式可用）
- 覆盖“远距切片”与至少 2 个高价值子切片（例如：小目标 + 遮挡、逆光/低照度、弯道/切入）
- 在既有评测协议上（mAP/NDS）实现：
  - 远距切片 mAP 或 recall 的 **稳定提升**（建议先设 2%–5% 绝对提升作为谈判目标，最终以你们当前基线水平校准）
  - 全量指标不明显退化（设守门阈值，例如全量 NDS 不低于基线 -0.2）

### 1.2 次 KPI（强烈建议完成）

- 把“增益归因”做扎实：给出 2 组消融（只改数据 / 数据+训练策略），并能解释收益来自哪些子切片
- 形成可交接资产：一键脚本 + 数据版本 + 报告模板 + 可视化（GUI 服务器优势）

## 2) 评测指标：在 mAP/NDS 之外，补上更科学的“切片增益证据链”

你们当前最信任 mAP/NDS 没问题，但它对“远距小目标”的敏感性不足，容易出现“全量没变化但远距变好/变坏”的误判。建议在不推翻现有体系的前提下，加 4 类辅助指标：

### 2.1 距离分桶（必须加）

把 0–100m 划分为可行动的 bins（先按 80m 目标，留出扩展）：

- 0–20m / 20–40m / 40–60m / 60–80m（短期）
- 80–100m（预留）

每个 bin 输出：mAP、Recall@固定FP、或 PR 曲线关键点（选择你们最容易落地的一个）。

### 2.2 小目标与可见性切片（强烈建议）

- 小目标定义：用 2D bbox 像素面积分桶（按相机）或 3D box 体积/高度分桶（按雷达/3D GT）
- 遮挡/截断：若你们标注里有 occlusion/crowding 字段则直接切片；没有就先用启发式 proxy（例如邻近物体密度/深度遮挡比例）

### 2.3 误检成本指标（更贴业务）

mAP 对误检的业务成本不敏感，建议至少加一个“可解释的误检率”指标：

- 例如 FP / km（按场景里程或时间归一化）
- 或 “关键区域 FP”（例如自车前方 ROI、车道内 ROI）

### 2.4 统计可靠性（最低成本的严谨性）

你们不需要上来就做复杂显著性检验，但建议做到：

- 对关键切片指标做 bootstrap 置信区间（哪怕只做 200–500 次重采样）
- 报告里明确：提升是否稳定（CI 是否跨 0）

## 3) 数据生成闭环的系统设计（你要搭的是“可控数据工厂”，不是单次仿真）

把系统拆为 5 层，层与层之间只用文件与配置通信（避免被某个平台锁死）：

1. **Scenario Spec（场景规格）**
   - 版本化 YAML/JSON：交通参与者类型与轨迹、初始相对距离、速度、遮挡物、天气/光照、传感器噪声/同步扰动（如允许）
2. **Asset Layer（资产层）**
   - 自车 3D 模型、HD map、传感器标定（内外参）、时间同步模型
3. **Generator（生成器）**
   - CARLA / Isaac Lab / alpaSim / cosmos 之一或多者
4. **Exporter（导出器：NuScenes 对齐）**
   - 关键产物：
     - `samples/` 6 路相机图像
     - `sweeps/` LiDAR sweep（多帧融合可用）
     - ego pose / timestamp 对齐
     - 标注：至少 3D box（服务 BEVFusion/CenterPoint），可选 2D box
5. **Evaluator + Miner（评测与挖掘）**
   - 对比基线 vs 增量
   - 自动挖掘失败子切片 → 反哺下一轮 Scenario Spec 采样策略

## 4) 仿真平台策略：CARLA/Isaac 先跑通闭环，同时为 alpaSim/cosmos 留“可替换接口”

### 4.1 平台定位（避免工程失控）

- CARLA：短期“最小闭环跑通”的首选（生态成熟、可控）
- Isaac Lab：对物理与可控交互更强，但自动驾驶传感器与地图生态需要评估工程成本
- alpaSim / cosmos：领导明确要测，建议你把它们定位为“第二阶段生成器”，前提是 Exporter/Evaluator 已稳定
- Alpamayo：不作为两个月主线训练目标；作为“最小 demo/接口层”更合理（见第 8 节）

### 4.2 cosmos 测试占位（你给出的命令形态）

你提到类似命令：

- `docker run -d --shm-size 96g --gups=all --ipc=host -p 8080:8080 cosmos-transfer1-carla`

注意：这里 `--gups` 可能是笔误，通常是 `--gpus=all`；以你们实际镜像/平台文档为准。

我建议你把 cosmos/alpaSim 的测试目标写成“能产出一条 NuScenes 可用序列”的硬标准，而不是“容器跑起来”。

## 5) 两个月里程碑（按周）——以 BEVFusion/CenterPoint 增益为中心

### Week 0：协议冻结（一天内可完成）

- 冻结评测协议：mAP/NDS + 距离分桶 + 目标类别列表 + 守门阈值
- 冻结数据格式：NuScenes 输出口径（时间戳、ego pose、sweeps 可用）
- 冻结定位连续性检查（沿用你的 GAP/TOLERANCE 规则）

交付物：一页“验收句 + 指标 + 切片定义 + 止损规则”。

### Week 1：最小链路跑通（端到端）

目标：用 CARLA 或 Isaac 任选其一，跑通

- `generate 10–20 sequences → export to NuScenes → 用现有 eval 脚本能读 → 出一份切片报告`

交付物：能在 GUI 上回放（相机/点云/3D boxes），并能输出距离分桶统计。

### Week 2：传感器与时间/位姿对齐做到“可用等级”

- 6cam+1lidar 内外参与时间同步模型对齐
- 导出 ego pose 与多帧 sweeps 可用（这是后续提升远距稳定性的关键）
- 加入你给出的 localization 连续性检查，确保导出数据不会在训练时引入隐性错配

交付物：NuScenes validator（自定义脚本也行）：
- 时间戳单调
- pose 连续
- sweeps 能回溯
- gap/tolerance 满足

### Week 3：目标切片的“场景库”成型（第一批 300–500）

围绕 80m 内重点类，做 3 类高价值场景模板（示例）：

- 远距小目标 + 遮挡（静态遮挡物/大车遮挡）
- 低照/逆光（傍晚、隧道口）
- 车速 20–42 km/h 下的相对运动与切入（目标横向速度扰动）

交付物：
- ≥300–500 sequences（带 `scenario_id` 与参数元数据）
- 自动生成的分布报表（距离/类别/速度/遮挡程度）

### Week 4：第一次混合训练验证（你不主训，但你要“驱动验证”）

- 交付给负责训练的同事：数据包 + 数据版本说明 + 训练建议（采样权重/混合比例/课程策略）
- 你负责：评测对比与归因

交付物：
- 一份对比报告：全量 + 距离分桶 + 关键类

### Week 5–6：闭环（失败挖掘 → 再生成 → 再验证）

- 从评测里自动挖 Top failure cases，聚合为 2–3 个子切片
- 生成第二批数据（累计 ≥1000）
- 做第二轮混合训练验证

交付物：
- 两轮增益曲线
- 消融：只加数据 vs 数据+采样/损失策略

### Week 7：把“数据工厂”工程化（可交接）

- 一键命令：`generate → export → validate → report`
- 结果落盘：每个版本都有数据 manifest、生成参数、覆盖统计

### Week 8：对内交付与资源申请

- 对内：明确 ROI（为啥这 1000 组数据值钱、下一步要多少算力/时间能换多少收益）
- 对外（可选）：抽象出不涉密的方法论与工具骨架

## 7) 风险与止损（按你现在的要求，重点防止时间对齐与格式坑）

- 传感器对齐是最大工程风险：如果 Week 2 仍无法稳定导出 NuScenes 可用序列，立刻降级：
  - 先只做 LiDAR + 关键 1–2 路相机（用于验证链路与指标）
  - 同时并行推进 6cam+1lidar 的完整对齐
- 若远距增益不显著：优先检查“分布是否真覆盖了失败子切片”，不要先改模型
- 严禁两个月内追求“NeRF 级照片真实感”作为硬目标：除非能证明它对远距/小目标识别有净收益

## 8) 最小 VLA/语言 Demo（你要的“时代接口”，但不分散主线）

定位：**语言 → 场景规格（Scenario Spec）** 的薄层，不做 RL、不训练大模型。

- 输入：自然语言（例如“生成雨天傍晚，右侧卡车切入，80m 内有锥桶”）
- 输出：结构化 `scenario_spec.yaml`
- 价值：让你的数据工厂“可交互、可展示、可复现”，同时服务主线

建议把它作为 Week 7 的展示增强项：主线闭环跑通后再做，避免抢占关键工程时间。
